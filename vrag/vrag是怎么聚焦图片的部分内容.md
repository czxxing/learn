### 代码层面的关键实现

**1. 模型输出裁剪指令** (`vrag_agent.py:105-110`)

```python
pattern = r'<(search|answer|bbox)>(.*?)</\1>'
match = re.search(pattern, response_content, re.DOTALL)
if match:
    action = match.group(1)  # 提取动作类型：'bbox'
    content = match.group(2) # 提取坐标：'[100, 200, 300, 400]'
```

**2. 裁剪并重新提交** (`vrag_agent.py:146-168`)

```python
elif action == 'bbox':
    # 1. 解析坐标
    bbox = json.loads(content)
    
    # 2. 坐标映射（解决缩放问题）
    input_w, input_h = self.image_input[-1].size
    raw_w, raw_h = self.image_raw[-1].size
    crop_region_bbox = [
        bbox[0] * raw_w / input_w,
        bbox[1] * raw_h / input_h,
        bbox[2] * raw_w / input_w,
        bbox[3] * raw_h / input_h
    ]
    
    # 3. 裁剪原始图片
    crop_region = self.image_raw[-1].crop(crop_region_bbox)
    
    # 4. 重新处理并发送给模型
    image_input, img_base64 = self.process_image(crop_region)
    user_content = [{
        'type': 'image_url',
        'image_url': {'url': img_base64}
    }]
    messages.append({'role': 'user', 'content': user_content})
```

### 核心设计思想

| 设计要点 | 说明 |
|---------|------|
| **模型自主决策** | 裁剪区域由VL模型根据分析需求决定，不是用户手动选择 |
| **迭代式分析** | 支持多轮裁剪（最多10步），逐步深入细节 |
| **上下文保留** | 每轮裁剪都基于之前的分析结果，形成推理链 |
| **坐标适配** | 自动处理图片缩放带来的坐标偏移问题 |

**简单来说**：模型像人类一样，先看整体，发现看不清的地方，就"放大"仔细看，直到找到答案。
