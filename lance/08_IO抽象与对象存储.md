# 第8章：IO 抽象与对象存储

## 概述

Lance 的 IO 抽象层统一了许多云存储后端的接口，支持 S3、Azure Blob、GCS。本章讨论抽象设计、处理方案、性能优化。

## ObjectStore 抽象接口

```rust
pub trait ObjectStore: Send + Sync + Debug {
    // 读取整个对象
    async fn get(&self, location: &Path) -> Result<Bytes>;
    
    // 读取范围
    async fn get_range(
        &self,
        location: &Path,
        range: Range<u64>,
    ) -> Result<Bytes>;
    
    // 读取多个范围
    async fn get_ranges(
        &self,
        location: &Path,
        ranges: Vec<Range<u64>>,
    ) -> Result<Vec<Bytes>>;
    
    // 写入对象
    async fn put(
        &self,
        location: &Path,
        bytes: Bytes,
    ) -> Result<()>;
    
    // 删除对象
    async fn delete(&self, location: &Path) -> Result<()>;
    
    // 列表对象
    async fn list(
        &self,
        prefix: &Path,
    ) -> Result<Vec<Path>>;
}
```

## 具体实现

### 本地文件系统

```rust
pub struct LocalFileSystemStore {
    root_path: PathBuf,
    cache: Option<Arc<Cache>>,
}

impl ObjectStore for LocalFileSystemStore {
    async fn get(&self, location: &Path) -> Result<Bytes> {
        let file_path = self.root_path.join(location.as_ref());
        let data = tokio::fs::read(&file_path).await?;
        Ok(Bytes::from(data))
    }
    
    async fn get_range(
        &self,
        location: &Path,
        range: Range<u64>,
    ) -> Result<Bytes> {
        let file_path = self.root_path.join(location.as_ref());
        let file = tokio::fs::File::open(&file_path).await?;
        let mut reader = tokio::io::BufReader::new(file);
        
        // 移动到指定位置
        reader.seek(SeekFrom::Start(range.start)).await?;
        
        let mut buf = vec![0u8; (range.end - range.start) as usize];
        reader.read_exact(&mut buf).await?;
        
        Ok(Bytes::from(buf))
    }
}
```

### AWS S3 存储

```rust
pub struct S3ObjectStore {
    client: aws_sdk_s3::Client,
    bucket: String,
    region: String,
}

impl ObjectStore for S3ObjectStore {
    async fn get(&self, location: &Path) -> Result<Bytes> {
        let resp = self.client
            .get_object()
            .bucket(&self.bucket)
            .key(location.as_ref())
            .send()
            .await?;
        
        Ok(resp.body.collect().await?.into_bytes())
    }
    
    async fn get_range(
        &self,
        location: &Path,
        range: Range<u64>,
    ) -> Result<Bytes> {
        let range_header = format!("bytes={}-{}", range.start, range.end - 1);
        
        let resp = self.client
            .get_object()
            .bucket(&self.bucket)
            .key(location.as_ref())
            .range(&range_header)
            .send()
            .await?;
        
        Ok(resp.body.collect().await?.into_bytes())
    }
}
```

## IO 调度器

```rust
pub struct IoScheduler {
    object_store: Arc<dyn ObjectStore>,
    batch_size: usize,
    max_requests: usize,
}

impl IoScheduler {
    // 批次读取范围
    pub async fn read_ranges(
        &self,
        location: &Path,
        mut ranges: Vec<Range<u64>>,
    ) -> Result<Vec<Bytes>> {
        // 排序和合并范围
        ranges.sort_by_key(|r| r.start);
        let coalesced = Self::coalesce_ranges(&ranges);
        
        let mut results = Vec::new();
        for range in coalesced {
            let data = self.object_store.get_range(location, range.clone()).await?;
            
            // 提取所需的每一段范围
            for original_range in &ranges {
                if original_range.start >= range.start && original_range.end <= range.end {
                    let offset = (original_range.start - range.start) as usize;
                    let len = (original_range.end - original_range.start) as usize;
                    results.push(data.slice(offset..offset + len));
                }
            }
        }
        
        Ok(results)
    }
    
    fn coalesce_ranges(ranges: &[Range<u64>]) -> Vec<Range<u64>> {
        if ranges.is_empty() {
            return Vec::new();
        }
        
        let mut coalesced = Vec::new();
        let mut current = ranges[0].clone();
        
        for range in &ranges[1..] {
            if range.start <= current.end {
                // 覆盖时合并为一段
                current.end = current.end.max(range.end);
            } else {
                coalesced.push(current);
                current = range.clone();
            }
        }
        coalesced.push(current);
        coalesced
    }
}
```

## 总结

IO 抽象机制：

1. **统一接口**：ObjectStore trait
2. **多云支持**：S3、Azure、GCS、本地文件
3. **调度优化**：范围合并并合并
4. **缓存一致性**：追赶缓存
5. **批次IO**：并列IO读取
